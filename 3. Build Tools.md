# Phase 3
## ILOs:
By the end of this phase, you will be able to:
- Compile and link multi-file C projects using GCC/G++
- Write Makefiles for automated build processes
- Create CMake configurations for cross-platform builds
- Use GDB to debug programs, set breakpoints, and inspect memory
- Analyze and fix memory leaks and segmentation faults
- Understand compilation stages (preprocessing, compilation, assembly, linking)
- Generate and read assembly code from C source
- Cross-compile for embedded targets (AVR, ARM)
## Why You Need This as an Embedded Software Engineer:
**Build Tools (Make/CMake)** are essential because embedded projects involve multiple source files, libraries, and hardware-specific configurations. You'll need to manage cross-compilation for different microcontroller targets, link against hardware abstraction layers, and automate testing - all impossible without proper build systems.

**GCC/G++** knowledge is crucial because you need to understand compiler optimizations, cross-compilation for ARM/AVR targets, and how to generate efficient code for resource-constrained devices. You'll use compiler flags to control code size, speed, and debugging symbols.

**GDB** is your primary debugging tool when hardware debuggers aren't available or when debugging complex issues. Embedded bugs are often harder to find (race conditions, timing issues, hardware interactions), and GDB helps you step through code, inspect registers, and analyze core dumps.
