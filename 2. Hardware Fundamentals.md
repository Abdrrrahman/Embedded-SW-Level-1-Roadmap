# Phase 2:
## ILOs:
By the end of this phase, you will be able to:
- Understand digital logic design and Boolean algebra
- Design and analyze combinational and sequential circuits
- Explain computer architecture components (CPU, memory, buses)
- Understand instruction execution cycles and pipelining
- Analyze memory hierarchy and cache organization
- Comprehend how high-level code translates to machine instructions
- Read and interpret timing diagrams
## Why You Need This as an Embedded Software Engineer:
**Logic Design** is fundamental because embedded engineers often interface directly with digital hardware. You need to understand how flip-flops, registers, and state machines work to design proper GPIO configurations, decode communication protocols, and troubleshoot hardware-software integration issues.

**Computer Architecture** knowledge is essential for understanding how your microcontroller executes instructions, manages interrupts, and accesses memory. This helps you write optimized code that respects CPU pipelines, cache behavior, and memory access patterns - critical for meeting real-time deadlines and power constraints. You'll understand why certain code patterns are faster, how DMA works, and how to optimize interrupt latency.

## Projects Ideas:
**Idea 1: Assembly to Machine Code Converter**
- Define custom instruction set (10-15 instructions)
- Create symbol_table to store labels and addresses
- Implement first_pass() to:
    - Scan for labels
    - Build symbol table
    - Calculate addresses
- Implement second_pass() to:
    - Convert mnemonics to opcodes
    - Resolve label references
    - Generate binary code
- Support directives: ORG (set origin), DB (define byte), DW (define word)
- Create assemble(source_file, output_file) function
- Create disassemble(binary_file) to convert back to assembly

**Idea 2: Simple CPU Simulator**
- Create 8-bit CPU with registers: PC (program counter), IR (instruction register), ACC (accumulator), SP (stack pointer)
- Implement memory array: uint8_t memory[256]
- Define instruction set: MOV, ADD, SUB, LOAD, STORE, JMP, JZ (jump if zero), HALT
- Create fetch() - read instruction from memory[PC]
- Create decode() - interpret instruction opcode and operands
- Create execute() - perform instruction operation
- Implement display_registers() to show all register values
- Implement display_memory(start, end) to show memory range
- Create step() function to execute one instruction cycle
