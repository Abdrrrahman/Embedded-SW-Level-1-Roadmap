# Smart Building Automation Node
## Project Overview
A fully self-contained building automation node that monitors environmental conditions, controls actuators, communicates with a master controller, runs multiple concurrent tasks under FreeRTOS, and is fully unit-tested and validated. This project touches every single skill you have built across all 6 phases.

---
## System Description
Your node will be installed in a room and will:
- Monitor temperature, humidity, light intensity, and motion
- Control a fan (PWM speed), lights (relay), and a door lock (relay)
- Display system status on an LCD
- Accept commands from a PC/Master via UART
- Log events to EEPROM
- Run all tasks concurrently under FreeRTOS
- Be fully unit-tested with mocked hardware drivers

---

## How Each Phase is Applied
**Phase 1**
- All firmware written in pure C with no HAL libraries
- A circular ring buffer (queue) used for UART command buffering
- A finite state machine (FSM) managing system operating modes (Normal, Alert, Lockdown)
- Bitwise operations for register manipulation throughout

**Phase 2**
- Understanding of GPIO electrical behavior, pull-up resistors, and signal timing
- Reading timing diagrams in sensor datasheets (DHT22, HC-SR04)
- Memory map awareness for register-level peripheral configuration
- Interrupt latency considerations for time-critical sensor readings

**Phase 3**
- Full CMake build system managing all source modules
- Separate build targets for embedded (AVR cross-compile) and host (native testing)
- GDB used for debugging on QEMU and live hardware via debugWIRE
- Compiler flags tuned for size optimization (-Os) on the embedded target

**Phase 4**
- DHT22 temperature & humidity sensor via One-Wire
- HC-SR04 ultrasonic sensor via Timer Input Capture (motion/proximity)
- LDR light sensor via ADC
- Fan speed control via Fast PWM
- Relay control for lights and door lock via GPIO
- LCD status display via I2C
- UART command interface with PC or master controller
- PIR motion sensor via external interrupt
- All events logged to EEPROM
- Watchdog Timer enabled for system reliability

**Phase 5**
- Task 1: Sensor Sampling Task: reads all sensors periodically
- Task 2: Actuator Control Task: drives fan, relays based on sensor thresholds
- Task 3: UART Command Task: receives and parses incoming commands
- Task 4: Display Task: updates LCD with current system state
- Task 5: Logger Task: writes events to EEPROM
- Queue used to pass sensor data between Sensor Task and Actuator/Display Tasks
- Mutex protecting shared EEPROM and LCD access
- Binary Semaphore triggering Logger Task from ISR on threshold breach
- Watchdog reset handled in Idle Hook

**Phase 6**
- All sensor drivers unit-tested on host machine using Unity
- Hardware registers mocked using FFF (Fake Function Framework)
- UART command parser fully tested with boundary and edge case inputs
- FSM state transitions covered with integration tests
- gcov/lcov used to measure and achieve >80% code coverage
- Static analysis run with Cppcheck before final submission
- TDD applied during UART parser and FSM development
